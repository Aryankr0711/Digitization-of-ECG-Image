║                         METRICS EXPLANATION                                  ║
╚══════════════════════════════════════════════════════════════════════════════╝

SNR (Signal-to-Noise Ratio) - in dB
────────────────────────────────────
Formula: 10 * log10(signal_power / noise_power)

Interpretation:
  • 25+ dB    = Excellent (almost perfect match)
  • 20-25 dB  = Very good (competition level)
  • 15-20 dB  = Good (acceptable)
  • 10-15 dB  = Fair (needs improvement)
  • < 10 dB   = Poor (significant errors)

Example: SNR = 23.45 dB means the signal is ~220x stronger than noise


RMSE (Root Mean Squared Error)
───────────────────────────────
Formula: sqrt(mean((predicted - actual)²))

Interpretation:
  • < 0.05    = Excellent
  • 0.05-0.1  = Very good
  • 0.1-0.2   = Good
  • 0.2-0.3   = Fair
  • > 0.3     = Poor

Example: RMSE = 0.0523 means average error is ~0.05 mV


MAE (Mean Absolute Error)
──────────────────────────
Formula: mean(|predicted - actual|)

Interpretation:
  • < 0.04    = Excellent
  • 0.04-0.08 = Very good
  • 0.08-0.15 = Good
  • 0.15-0.25 = Fair
  • > 0.25    = Poor

Example: MAE = 0.0412 means average absolute error is ~0.04 mV


MSE (Mean Squared Error)
─────────────────────────
Formula: mean((predicted - actual)²)

Interpretation:
  • < 0.005   = Excellent
  • 0.005-0.01 = Very good
  • 0.01-0.04 = Good
  • 0.04-0.09 = Fair
  • > 0.09    = Poor

Example: MSE = 0.002734 means squared error is very low
